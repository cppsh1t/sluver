# sluver 构想 v0.1.0

一款 AI 编写小说的桌面软件。用户在结构化的世界观基础上，借助 AI 生成小说内容。本文件是丢给 AI 让其一步步实现完整产品的描述文档。

---

## 一、技术栈

- **Tauri 2** + **React** + **TypeScript** + **shadcn/ui**
- **Vercel AI SDK**：在前端运行，不需要 Node 服务端。LLM 调用、流式、结构化输出全部在前端 webview 内完成
- **Rust 后端**：所有数据操作（读写数据库）放在 Rust 端，以 Tauri command 的形式向前端 AI agent 暴露为工具
- **SQLite**（带 FTS5 全文检索）
- **模型**：当前写死 **DeepSeek V4 Pro**，后期支持自定义 provider

> 关键：LLM 调用发生在前端，所以流式/结构化输出不过 Tauri IPC。Rust 只在 agent 调用数据工具时介入。

## 二、概念模型

### 2.1 顶层结构

应用顶层分两类对象：

- **世界（World）**：一个完整的设定宇宙，是创作的工作空间
- **配置（Config）**：应用级配置（模型、API key、外观等）

### 2.2 世界下的元素

一个世界下包含以下**互相独立的元素类型**，它们之间通过**引用**关联（同一个角色可以在不同事件、不同小说中被引用）：

| 元素                   | 说明                                                                    |
| ---------------------- | ----------------------------------------------------------------------- |
| **小说（Novel）**      | 一部完整的小说作品                                                      |
| **角色（Character）**  | 世界中的人物                                                            |
| **地点（Location）**   | 世界中的场所                                                            |
| **物品（Item）**       | 世界中的道具、物件                                                      |
| **设定（Setting）**    | 世界观补充说明（势力、魔法体系、历史背景等）                            |
| **事件（Event）**      | 记录对世界有影响的事件，包含参与角色、地点、时间等                      |
| **时间线（Timeline）** | 以事件为基础节点展示世界/小说的时间发展，点击节点可跳转相关的人物或事件 |

> 地点、物品、设定本质是补充说明类元素。
>
> **v0.1.0 不做时间线 UI**，但事件数据可以先存。

### 2.3 角色 phase

角色不是单一固定状态。一个角色包含**多段 phase**，每段 phase 对应角色的一个时期，描述该时期下的：

- 身份、处境
- 性格、心理状态
- 关系变化
- 能力、外观等状态变化

phase 之间有时序关系，由事件触发转换。场景在引用角色时，实际上引用的是角色在某个 phase 下的状态。

### 2.4 小说的结构化存储

小说**不是以纯文本存储**，而是结构化的树：

```

章（Chapter）
├── 章节梗概
└── 场景（Scene）  ← 叶子单元，AI 生成的最小单位
    ├── 场景梗概（来自大纲）
    ├── 具体内容（AI 生成的正文）
    └── 关联实体引用（角色/地点/事件等）
```

**叶子单元叫"场景"**。一章由多个场景组成。AI 每次生成一个场景的内容。每个场景可以关联本章出场的角色（及其 phase）、地点、涉及的事件等。

## 三、界面与交互

### 3.1 总体导航

应用采用侧边栏 + 主内容区的布局：

- **侧边栏**：世界列表 / 当前世界内的元素导航（小说、角色、地点、物品、设定、事件）
- **主内容区**：根据所选内容展示对应视图

### 3.2 世界视图

选中一个世界后，主内容区展示世界内的所有元素，按类型分组。可以新建/编辑/删除任意类型的元素。

### 3.3 元素编辑

每种元素都有自己的编辑表单：

- **角色**：基础信息 + 多段 phase 编辑 + 自由备注
- **地点/物品/设定**：名称 + 内容描述 + 自由备注
- **事件**：名称 + 时间 + 参与角色（引用）+ 地点（引用）+ 描述

### 3.4 小说视图

选中一本小说后，展示小说的结构化树：梗概、大纲、章节列表、每章下的场景列表。可以：

- 编辑小说梗概、整体大纲
- 新建章、编辑章节梗概
- 在章下新建场景、给场景写梗概
- 点击场景进入 AI 编写页或查看场景正文

### 3.5 AI 编写页（核心）

进入一个世界 → 进入一本小说 → 新建/选择一章 → 进入某个场景的 AI 编写页。这是产品的核心界面。

**布局**：左侧元素面板 + 右侧 AI 对话区。

**左侧元素面板**：展示与当前场景相关的元素（角色、地点、事件等）。这些元素和右侧 AI 对话**共享上下文**——AI 在对话中查到的元素会自动出现在左侧面板。用户也可以手动从世界库里挑选元素加入面板，作为 AI 的上下文。面板中的每个元素都**支持用户删除**。

**右侧 AI 对话区**：类似 AI 聊天界面。v0.1.0 提供 **write agent**：

- 用户描述想要这个场景怎么写，或给出指示
- write agent 自主调用工具从数据库里查询它需要的上下文（角色当前 phase、前序场景内容、相关设定、地点描述、涉及的事件等）
- agent 把查到的关键元素加入左侧面板（共享上下文）
- 一旦用户点击「撰写」，agent 用 **Vercel AI SDK 的结构化输出（`generateObject` + schema）** 生成该场景的完整对象：场景正文内容 + 涉及的实体引用 + 元数据
- 生成完成后，场景写入数据库，关联的实体引用自动建立

### 3.6 撰写过程

用户点击「撰写」后，对话页切换为撰写视图，展示 AI 正在生成场景内容。用户可以中途停止，或等待完成。完成后回到场景视图，可以看到生成的正文和自动建立的实体引用。

## 四、AI 工作方式

### 4.1 tool-using agent

write agent 是一个**使用工具的 agent**：它通过调用 Rust 端暴露的 Tauri command 来查询数据库，自主决定需要哪些上下文。

**不做 RAG / 向量检索**。信息检索完全靠工具调用（基于 FTS5 全文搜索 + 结构化查询）。理由：RAG 的 embedding 持续写入维护成本高，工具检索对当前需求够用。

### 4.2 Rust 端暴露的工具（给 agent 用）

Rust 端把数据操作封装为 Tauri command，前端将其注册为 agent 可调用的工具。大致分类：

- **实体查询**：按 id 取角色（含 phase）/地点/事件/设定；按条件列表/过滤
- **场景上下文**：取前序 N 个场景正文、章节梗概、小说梗概与大纲
- **全文检索**：FTS5 跨 lore 库搜索（按关键词找角色/设定/事件）
- **写入**：写场景内容、建立场景与实体的引用边

具体工具 schema 在实现时细化。

### 4.3 结构化输出

场景生成用 Vercel AI SDK 的 `generateObject` + zod schema，输出一个结构化对象，包括：

- 场景正文（小说叙述文本）
- 本场景出场的角色（引用 id + 使用的 phase）
- 涉及的地点/物品（引用 id）
- 涉及的事件（引用 id）
- 其他元数据

### 4.4 元素自动关联与可删除

agent 在查询和生成过程中发现的元素，会自动加入左侧面板和场景的引用列表。**所有自动加入的元素都支持用户删除**——不是不可逆的自动写入。这是 agent 通过工具从数据库查出来的真实元素，不是从生成文本里抽取或幻觉。

## 五、v0.1.0 范围

本版本是验证核心闭环的原型。明确**做**与**不做**：

### 做

- 完整的概念模型：世界 / 小说 / 角色（含 phase）/ 地点 / 物品 / 设定 / 事件
- 小说的结构化存储（小说 → 章 → 场景）
- 元素的 CRUD + 引用关系
- AI 编写页：左侧元素面板 + 右侧对话区
- write agent：工具调用查上下文 + 结构化生成场景 + 元素自动关联
- 模型写死 DeepSeek V4 Pro

### 不做（后续版本）

- 时间线 UI（事件数据可以先存）
- 版本历史 / 修订记录
- 导出
- 编辑 / 重新生成的迭代循环
- editor / 一致性校验 agent
- RAG / 向量检索
- 自定义模型 provider
